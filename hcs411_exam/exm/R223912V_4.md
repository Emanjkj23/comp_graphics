# Qustion 4 (a)
- Given:
- 3D vertex x = (x, y, z, 1)ᵀ in world coordinates
- Modelview matrix M (world to eye space)
- Perspective matrix P (projection transformation)
We do the following:

Modelview transformation (eye space):
v' = M · x
Transforms vertex to camera-relative coordinates

Perspective transformation (clip space):
v'' = P · v'
Applies perspective projection; result has homogeneous coordinate w''

Perspective division (normalized device coordinates):
v_ndc = v'' / w''
Divides x, y, z by w to get final NDC coordinates in range [-1, 1]

```cpp
void reshape(int w, int h) {
    // Use ACTUAL window dimensions, not hardcoded values
    glViewport(0, 0, w, h);
    glMatrixMode(GL_PROJECTION);
    glLoadIdentity();
    // Use ACTUAL aspect ratio to prevent distortion
    gluPerspective(45.0, (double)w / (double)h, 1.0, 100.0);
    glMatrixMode(GL_MODELVIEW);
}
```

**Why your code was incorrect:**
- `glViewport(0, 0, 800, 600)` was hardcoded—won't adapt when window resizes
- Aspect ratio `800/600` was hardcoded—causes distortion if user resizes to different proportions

**How it maintains proportions:**
- `glViewport(0, 0, w, h)` — Maps drawing area to actual pixel dimensions
- `(double)w / (double)h` — Adjusts projection's aspect ratio dynamically
- Together, they scale the scene to fill any window size without stretching
